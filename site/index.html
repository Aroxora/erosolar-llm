1. The Current Honest Small-LLM Pipeline

erosolar trains successively stronger small models (~2M–100M parameters today) using a closed, measurable loop that reports only numbers that were actually run.

Core Components

  • Model: Small decoder-only transformer with Infini-Attention — compressive memory for longer effective context without quadratic cost blowup at small scale.
  • Tokenizer breakthrough: Word tokenizer → 0% held-out generalization. Char tokenizer + length-matched nonce copy-augmentation induced a copy/induction head, jumping validity 0% → 53% → ~89%. Direct application of mechanistic interpretability findings on induction heads.
  • Data: 229 wholesome qualities, license-clean, template-composed self-generated — no teacher model distillation. Every sample is Python-checkable for validity + appropriateness.

The Generational Upgrade Pipeline

  1. Analyze losers: Bottom 25% by the internal master scalar (average pairwise similarity of Chain-of-Thought embeddings; lower = more diverse reasoning).
  2. Generate variants: Complexity-escalated "friend" samples targeting the diagnosed gaps.
  3. Grounded verification (before any sample enters training): code execution, SymPy, Z3 constraints, multi-sample self-consistency.
  4. Train & measure: Fine-tune, recompute master scalar + external benchmarks.
  5. Quality gate: Master scalar must improve; no regressions on validity (≥0.95), appropriateness (≥0.95), coverage (1.0), distinctness, or LLM-judge scores (overall ≥0.85, wholesomeness ≥0.9). See test_invariants.py.

The pipeline is fully reproducible with ./run.sh (or --generalize for the char+copy 89% recipe).

2. The Additive Agent Stack

Nothing touches the small core. A separate layer on top:

  • Finetune: QLoRA SFT → DPO along a smallest-first model ladder — from the Qwen3-0.6B floor (smallest confirmed general-purpose + Apache-2.0) up through Qwen3-30B-A3B (MoE) to the Qwen3-32B final, on Lambda H100. See /pipeline for the cited ladder + fine-tune cost by size.
  • Runtime: Model-agnostic plan → act → observe → reflect loop with durable memory, tool use, and context compaction. Works against any OpenAI-compatible endpoint.
  • Serving: vLLM (base model) + agent_server behind OpenAI-compatible API.
  • Integrations: Tavily (search with quota handling) + DeepSeek (synthesis). Same integrations power the model-landscape/ auto-updater.
  • Eval: Agentic task suite + lm-eval slate.

This is why a tiny honest core can still be useful: the long-horizon behavior lives in the runtime + tools, not purely in the weights.

3. The Upgrade Pipeline & Future Directions

The current loop is deliberately architecture- and scale-agnostic in its data generation and verification. The same "loser analysis + grounded verification + quality gate" can drive improvement at larger scales.

Current Strengths at Small Scale

  • Copy/induction heads (via data) are extremely high-leverage — parametric memory is expensive at small scale.
  • Grounded verification acts as a strong regularizer and honesty enforcer.
  • The master scalar steers diversity without gaming external benchmarks.

Upgrade Vectors

Drawn from the living architecture-roadmap/ and model-landscape/:

  • Bigger models: Apply identical pipeline to 100M+ then 1B+ scales. Small model as fast proposer/verifier in speculative or tree-search setups.
  • Better architecture: Hybrids (Mamba-2 / SSM blocks interleaved with attention), sub-quadratic attention variants, improved induction mechanisms. Mixture-of-Experts is not useful below ~hundreds of millions of active parameters.
  • Test-time compute: Even the 14M model benefits from extra thinking (multi-sample + judge, tree search, code execution feedback). Small models + heavy verification can outperform naive larger models on narrow tasks.
  • RAG & tools: Small models are knowledge-poor by design — offload to retrieval and the agent runtime.
  • Quantization & edge: Design for 4-bit / 8-bit from the start. Linear decode and small KV/state matter here.
  • Distillation & self-improvement: Use the honest small model + verifiers to generate higher-quality data. Keep everything license-clean and verifiable.
  • Model-landscape awareness: model-landscape/update.py keeps an agentically refreshed picture of frontier models, preventing over- or under-estimation of what "small" can do.

Concrete Next Steps

  • Run full pipeline at larger scale (more params, more data, same verification).
  • Integrate small core as fast proposer/verifier inside agent runtime.
  • Experiment with hybrid blocks in the core transformer.
  • Close the loop: use agent to propose better training distributions or verification oracles.

All numbers in this document were measured. The invariants in test_invariants.py will loudly fail if that stops being true.
